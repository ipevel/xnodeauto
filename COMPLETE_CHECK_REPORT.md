# XnodeAuto 项目全面检查完整报告
=============================================

=============================================
一、严重问题（必须立即修复）
=============================================

1. install.sh - 参数解析严重Bug
----------------------------------
位置: 第118-119行、126-127行
问题: ADMIN_PASSWORD 和 PANEL_TOKEN 被硬编码为 "***"，命令行参数完全失效
代码:
  --admin-password) ADMIN_PASSWORD="***"; shift 2 ;;
  --panel-token)    PANEL_TOKEN="***";    shift 2 ;;
影响: 使用 --admin-password 和 --panel-token 参数时，实际值被丢弃
修复方案:
  --admin-password) ADMIN_PASSWORD="$2"; shift 2 ;;
  --panel-token)    PANEL_TOKEN="$2";    shift 2 ;;

2. xnode.sh - Authorization Header 截断错误
-----------------------------------------------
位置: 第159行
问题: Authorization header 被截断，使用硬编码 "***" 且缺少闭合引号
代码: -H "Authorization: Bearer *** \
影响: API 请求永远失败，无法从面板获取节点名称
修复方案: -H "Authorization: Bearer ${panel_token}" \

3. update-xboard-node.sh - 版本比较逻辑严重错误
-----------------------------------------------
位置: 第51行
问题: 仅做字符串相等比较，不做语义化版本比较
代码:
if [ "$CURRENT_VERSION" = "$LATEST_VERSION" ] || [ "$CURRENT_VERSION" = "v$LATEST_VERSION" ] || [ "v$CURRENT_VERSION" = "$LATEST_VERSION" ]; then
影响: v1.0.10 会被认为比 v1.0.2 旧（字符串比较）
修复方案: 使用语义化版本比较函数

4. update-xboard-node.sh - 备份文件清理无效
-------------------------------------------
位置: 第88行、113行
问题: 备份文件名是 xboard-node.bak，但查找的是 xboard-node.bak.*
影响: 备份文件永远不会被自动清理
修复方案: 统一备份文件名带时间戳，或修改 find 命令

=============================================
二、高优先级问题
=============================================

5. xnode.sh - version_ge 函数逻辑错误
-----------------------------------------
位置: 第65-68行
问题: 版本比较函数返回结果与预期相反
当前代码:
version_ge() {
    test "$(printf '%s\n' "$1" "$2" | sort -V | head -n 1)" != "$1" -o "$1" = "$2"
}
修复方案:
version_ge() {
    test "$(printf '%s\n' "$@" | sort -V | head -n 1)" = "$2"
}

6. upgrade.sh - set -e 导致的意外退出
-------------------------------------
位置: 第223-224行
问题: 即使有 || echo "未知"，set -e 仍可能导致脚本退出
修复方案: 添加 || true 防止意外退出

7. 所有脚本 - 缺少下载文件校验
---------------------------------
问题: 所有通过 wget/curl 下载的文件都没有进行 SHA256 校验
风险: 中间人攻击、下载被篡改的文件
修复方案: 增加下载后验证文件哈希

8. 所有脚本 - 直接覆盖关键系统文件
-------------------------------------
问题: 直接下载到 /usr/local/bin/，下载失败会留下损坏文件
修复方案: 先下载到临时目录，校验通过后再移动

=============================================
三、中优先级问题
=============================================

9. xnode.sh - 双斜杠路径拼写错误
-------------------------------------
位置: 第87行
问题: cat //proc/version 有多余的斜杠
修复方案: 改为 /proc/version

10. 所有脚本 - JSON 解析方式脆弱
----------------------------------
位置: install.sh 180行、217行，upgrade.sh 106行、114行，update-xboard-node.sh 32行
问题: 使用 grep + cut 解析 JSON，格式变化就会失效
修复方案: 优先使用 jq，保留 grep + cut 作为后备

11. upgrade.sh - 通配符循环可能处理不存在的文件
-------------------------------------------------
位置: 第189行
问题: 没有匹配文件时，shell 会将通配符本身传递给循环
修复方案: 使用 shopt -s nullglob

12. xnode.sh - sed 命令中的变量未转义
-----------------------------------------
位置: 第697、700、772行
问题: 直接在 sed 中使用变量，可能包含特殊字符
风险: 命令注入或 sed 错误
修复方案: 对变量进行转义，或使用 yq

=============================================
四、低优先级问题
=============================================

13. xnode.sh - 多处 show_menu 递归调用
-------------------------------------------
位置: 多处函数末尾调用 show_menu
问题: 可能导致栈溢出
修复方案: 改为循环结构

14. xnode.sh - Alpine Linux 支持不完整
-------------------------------------------
问题: 多处假设 systemd 存在
修复方案: 增加 OpenRC 支持

15. 所有脚本 - 错误处理不完善
---------------------------------
问题: 很多命令使用 2>/dev/null || true 静默忽略错误
修复方案: 只对预期可能失败的命令使用

16. 所有脚本 - 缺少依赖检查
-------------------------------
问题: 脚本依赖 curl、wget、systemctl 等工具，但没有预先检查
修复方案: 在脚本开始处检查所有必需的依赖
