## 技术架构

### 核心组件
```
/usr/local/bin/
├── xboard-node          # 节点程序（来自 cedar2025/Xboard-Node）
├── sync-nodes           # 同步程序（自研，Go 语言）
├── update-xboard-node.sh # 自动更新脚本
└── xnode                # 管理脚本（Bash）

/etc/xboard-node/
├── sync.yml             # 主配置文件
├── node_alias.yml       # 节点别名配置
├── .token               # Token 缓存
└── *.yml                # 各节点配置文件

/etc/systemd/system/
├── xboard-node@.service # 节点服务模板
├── sync-nodes.service   # 同步服务
├── sync-nodes.timer     # 同步定时器（每分钟）
├── update-xboard-node.service # 自动更新服务
└── update-xboard-node.timer   # 自动更新定时器（每天凌晨3点）
```

### 工作流程

#### 安装流程
1. 检测系统架构
2. 安装依赖（wget、curl）
3. 下载 xboard-node 二进制
4. 下载 sync-nodes 二进制
5. 创建配置目录
6. 安装 systemd 服务
7. 安装更新脚本
8. 安装管理脚本
9. 配置引导（面板地址、后台路径、邮箱、密码、Token）
10. 选择同步方式（自动/手动）
11. 执行首次同步
12. 启动定时服务

#### 同步流程
1. 读取配置文件（sync.yml）
2. 获取本机 IP（通过 ip.sb）
3. 登录面板获取 Token
4. 获取面板节点列表
5. 确定需要管理的节点：
   - 如果配置了 manual_node_ids：使用手动指定的节点
   - 否则：通过 IP 自动匹配
   - 如果自动匹配到 ≥2 个节点：停止并提示使用手动模式
6. 对比当前配置：
   - 需要启动的节点：创建配置文件、启动服务
   - 需要停止的节点：停止服务、删除配置文件
7. 输出同步结果

## 安全策略

### 自动匹配规则
| 检测结果 | 行为 | 原因 |
|----------|------|------|
| 0 个节点 | ✅ 正常运行 | 无节点需要启动 |
| 1 个节点 | ✅ 自动启动 | 明确的单一节点 |
| ≥2 个节点 | ❌ 停止并提示 | 防止中转机场景误操作 |

### 中转机场景
对于中转机/CDN场景，**必须**使用手动模式：

```bash
# 安装时选择手动模式
# 或手动编辑配置文件添加 manual_node_ids

# 推荐使用命令添加
xnode add-node <节点ID>
```
